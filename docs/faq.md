# 常见问题与故障排除

---

## 安装与环境

### Q: 安装后 `dualign` 命令找不到？

A: 确保虚拟环境已激活。运行 `pip install -e .` 后重新打开终端。使用 `dualign --version` 验证安装。

### Q: `pip install -e .` 报错？

A: 检查网络和虚拟环境。权限问题可改用 `pip install --user -e .`。Windows 用户注意 PowerShell 执行策略。

### Q: 是否支持 GPU 加速？

A: 嵌入编码依赖外部服务（Ollama/LM Studio），是否使用 GPU 取决于该服务的配置。Dualign 自身不直接调用 GPU。

---

## 安装后启动闪退或报错？

### Q: GUI 启动后一片空白或立即闪退？

A: 终端运行 `dualign gui` 查看完整报错。常见原因：

- **PySide6 未正确安装**：`pip install PySide6>=6.5`
- **显卡驱动不兼容**：更新显卡驱动
- **端口冲突**：Ollama 默认 11434 端口被占用时，设环境变量 `DUALIGN_OLLAMA_URL`，或在设置面板中修改 API 地址

## GUI 操作

### Q: 不小心用 `Ctrl+B` 把左侧面板收起来了，怎么恢复？

A: 菜单栏 **视图 → 显示/隐藏侧边栏**（快捷键 `Ctrl+B` 再次切换）。也可在 **视图** 菜单中单独勾选「审校面板」「文件管理」恢复指定面板。快捷键 `Ctrl+Alt+B` 切换右侧辅助面板。

### Q: 底部的 AI 建议面板不见了？

A: 三种可能：

1. **被折叠了**：点击底部栏的 ▼ 按钮，或快捷键 `Ctrl+J` 切换
2. **欢迎页锁定**：加载文件后自动解锁
3. **AI 不可用且无建议**：符合条件时自动展开

底部栏高度支持 30%/40%/50% 三档，拖拽手柄可切换。

### Q: 点击「预览」模式后表格变了，数据没丢吧？

A: 没丢。**校订**模式（7 列）和**预览**模式（4 列）只是同一数据的两种视图——校订模式显示完整操作信息，预览模式聚焦于「行号 + 评分 + 原文 + 译文」。通过 StatusBar 的按钮可随时切换，数据完整保留。

### Q: 筛选后表格空白了？

A: 筛选面板有**两个维度**：异常类型（4 类）和处理状态（4 种）。同一维度内多选为 OR，跨维度为 AND（可在筛选面板底部切换为 OR）。若全部勾选后无结果：

1. 检查是否真无匹配——试试**全取消**然后只勾一项
2. 右下角「显示已处理」checkbox 是否未勾选（AI 建议面板的过滤独立于主表筛选）

### Q: 右键表格能做什么？

A: 表格右键菜单提供丰富的快捷操作，包括：

- **复制格式**：Markdown 表格 / TSV（制表符分隔）
- **编辑操作**：合并 `[M]`、拆分 `[S]`、校订 `[E]`、删除 `[D]`
- **批量操作**：多选后「合并选中」「校订选中」「删除选中」
- **审核**：审核通过 [OK]、标记异常 [F]

右键操作是完成日常校订最快的方式。

### Q: 左侧审校面板的章节怎么折叠了？

A: 审校面板各功能节（如章节导航、操作按钮、低分统计等）均为可折叠设计。点击节标题栏（带 ▼/▶ 箭头）可展开或收起。

### Q: 文件从欢迎页拖不进工作区？

A: 工作区的原文/译文输入框支持**从文件管理器拖拽文件**放入（仅 `.md` 和 `.txt`）。欢迎页本身不支持拖拽，请先打开文件对，或使用欢迎页的「打开文件对」按钮。

## 对齐结果

### Q: 对齐后全是 1:0 或 0:1？

A: 最常见原因：① 嵌入后端未就绪（运行 `dualign check` 检查，欢迎页指示灯应为绿色）；② 原文和译文不是行级平行文档（段落级也可接受，对齐器将每行视为一个完整语料单元）。运行 `dualign check` 查看质量门控报告。

### Q: 质量门控显示 "unreliable"？

A: 真锚点密度低于 60%，表示原文和译文的结构对应关系偏弱。常见于段落级文本或非平行文档。这是启发式警告——建议检查输出，但不必恐慌。

### Q: 为什么有些评分很低但内容其实是正确的？

A: 约 35% 的短句因稀有专有名词或标点差异导致评分偏低。评分是参考信号——**连续多行低分**才值得警惕，单行低分通常无需处理。

### Q: 切换嵌入模型后需要重新编码吗？

A: 是。缓存键包含模型名和 Instruction 哈希，自动失效。首次编码会慢（重新生成缓存），之后恢复正常。

### Q: 撤销（Ctrl+Z）能撤销到什么程度？

A: 撤销栈保存**最近 50 步操作**，仅限于当前会话。关闭文件对后撤销栈清空。每次操作后系统会自动保存修复日志到 `report.json`，重开文件对可以恢复进度——但撤销栈本身不持久化。

## AI 审校

### Q: AI 审校按钮不可用？

A: 需配置 DeepSeek API Key。两种方式：

- **环境变量**：`$env:DEEPSEEK_API_KEY = "sk-xxx"`（PowerShell）
- **GUI 设置**：菜单 **设置 → 模型与 Agent 配置** 填入 API Key

底部栏的 AI 指示灯应为绿色。

### Q: AI 审校运行时卡住不动？

A: DeepSeek API 通常 10-30 秒响应。超时 60 秒请检查：

1. API Key 余额
2. 网络连接（是否需代理）
3. 终端有无错误输出

### Q: AI 审校的结果可靠吗？在花钱吗？

A: AI 擅长判断语义对应关系，但不建议替代人工终审——特别是创意性译文。Dualign 使用 DeepSeek API（约 $0.14/百万 tokens），每次校订大约几分钱。建议逐条审查 AI 建议，把它作为"第一遍粗筛"。

### Q: AI 建议面板的「显示已处理」是什么？

A: 勾选后，已经手动确认/修改过的 AI 建议也会显示。取消勾选则只显示待处理的建议——如果 AI 建议突然少了很多，先检查这个 checkbox。

## 导出与固化

### Q: 固化（Promote）是什么？

A: 将修复后的文件（`repaired/` 目录）**覆盖回原始文件**。这是一个**破坏性操作**——原始文件会被备份为 `.bak`。固化后需要重新对齐和校订。在 GUI 操作面板点击「⬆ 固化修复」前，请确认已导出所需结果。

### Q: 导出的文件编码？

A: 所有输出均为 **UTF-8**。原始文件也需为 UTF-8，否则显示为乱码。

### Q: 同时处理多个文件怎么做？

A: 使用**工作区**——在文件管理面板中将多个文件加入队列，批量对齐。也可用 CLI：`dualign align -s A.md -t B.md -o ./output`。

## 数据与缓存

### Q: 配置和缓存存在哪里？

| 内容     | 路径                                           |
| -------- | ---------------------------------------------- |
| GUI 配置 | `%LOCALAPPDATA%/dualign/cache/gui_config.json` |
| 嵌入缓存 | `%LOCALAPPDATA%/dualign/cache/embedding/`      |
| 模型配置 | `~/.dualign/providers.json`                    |
| 修复报告 | 输出目录的 `report.json`                       |

### Q: 想重置所有 GUI 设置？

A: 菜单 **设置 → 恢复默认设置**，不影响模型/Agent 配置。如需完全清除，可删除 `%LOCALAPPDATA%/dualign/cache/` 下的配置文件。

## 已知限制

- **N:M 对齐不支持**：系统不产生 N:M（如 2:2、3:2）类型的对齐对
- **段落重排不支持**：假设原文和译文保持单调顺序（H1 约束）
- **仅行级（段落级）对齐**：不支持词级或短语级对齐。对齐器将每行视为一个完整语料单元，仅在"拆分"操作时基于标点符号拆分为句子级后重新对齐
- **嵌入模型需求**：需要支持文本编码的模型（sentence-transformers 类）
- **语种覆盖**：核心算法语言无关，但语言杂糅检测针对中→英场景设计，其他语种对需调整
- **AI 审校实验性**：Tool-calling 代理在复杂场景下可能做出错误决定，建议逐条审查 AI 建议
- **撤销栈内存化**：关闭文件对后撤销/重做历史清空（修复日志仍保存于 report.json）
