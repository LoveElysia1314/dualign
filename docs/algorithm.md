# 对齐算法：思想、数学与工程

> 面向算法贡献者和技术研究用户。需了解嵌入编码和动态规划基础。

---

## 一、问题本质

给定原文句子序列 `S[1..n]` 和译文句子序列 `T[1..m]`，行数 n 与 m 无先验等式约束。

形式化目标：找到一组对齐操作，每个操作 (src_indices, tgt_indices, score) 将原文的一个子序列映射到译文的一个子序列，满足：

1. **完全覆盖**：每个原文行和译文行都恰好属于一个操作。
2. **单调有序**：操作之间的原文行序和译文行序互不交叉。
3. **最大化总分**：所有操作的评分之和尽可能高。

**操作类型**：

| 类型 | 含义           | 原文行数 | 译文行数 |
| ---- | -------------- | -------- | -------- |
| 1:1  | 正常对应       | 1        | 1        |
| N:1  | 多行合并为一行 | N (≥2)   | 1        |
| 1:M  | 一行拆分为多行 | 1        | M (≥2)   |
| N:0  | 原文缺失译文   | N (≥1)   | 0        |
| 0:M  | 译文多余       | 0        | M (≥1)   |

> 系统**不产生 N:M 类型**（如 2:2），因为系统不引入自己不能保证的新信息。

## 二、核心思想：先锚点，再填充

在不确定中找到一组**可信的参照系**（锚点），然后用它们将问题分解为独立可解的小段。

- **真锚点**（True Anchors）是递归 LIS 双边互惠确认的高置信度 1:1 对。源侧认为译侧在自己的邻域内足够好，译侧也认为源侧足够好。
- **赝锚点**（Pseudo-Anchors）由受限 DP 以最大化局部总分的方式补全，不要求互惠性，仅作分解工具。
- 真锚点将长序列切成一串独立段，全局最优退化为逐段独立最优的拼接。

**为什么不直接用全量 DP？** 全量 DP 搜索空间 O(n × m)，且引入 N:1/1:M 合并时候选集编码成本随文档长度平方级增长。真锚点将问题降为 O(K × θ)，其中 K 为真锚点数（通常 K << min(n, m)），θ 为单次合并最大跨度（默认 20）。

## 三、Phase 1 → 5 流水线

### 3.1 Phase 1 — 递归双边 LIS 搜索（真锚点）

**核心洞察**：在全局范围内，正确配对 (i, j) 可能因同行其他高分候选压制 top-1，被信任余量否决。但将矩阵按已有真锚点分段后，子段内的竞争对手减少，该配对的 top-1 排名上升，可能重新通过双边检查。

因此真锚点搜索是**递归迭代**的——每轮至少增加 1 个锚点，直到收敛。

**双边信任余量**：信任余量是一个动态阈值，随 top-1 得分线性变化：

$$
\text{margin}(s) = 0.10 \times s - 0.05, \quad s \in [0.6, 1.0]
$$

候选配对 (i, j) 需同时满足三个条件：

1. 最低质量门控：余弦相似度 C(i, j) ≥ 0.6
2. 源侧信任检查：C(i, j) ≥ top1_src[i] - margin(top1_src[i])
3. 译侧信任检查：C(i, j) ≥ top1_tgt[j] - margin(top1_tgt[j])

**margin 对照表**：

| top-1 得分 | margin 值 | 放行区间     | 含义           |
| ---------- | --------- | ------------ | -------------- |
| 1.0        | 0.05      | [0.95, 1.0]  | 高分宽容       |
| 0.8        | 0.03      | [0.77, 0.80] | 中等，适度收紧 |
| 0.6        | 0.01      | [0.59, 0.60] | 低分严苛       |

margin 呈正斜率——得分越高越宽容。双侧过滤天然杜绝错配级联。

### 3.2 Phase 2 — 受限 DP 补全赝锚点

Phase 1 收敛后，在所有未被覆盖的双侧区域一次性运行受限 DP（仅允许 1:1、1:0、0:1 三种类型），取其 1:1 对作为赝锚点，不再递归。

### 3.3 Phase 3 — 全局枚举合法合并组合

从**全体 1:1 对**（真锚点 + 赝锚点）出发，将行分为两类：

- **基准行**：属于某个 1:1 对的行
- **自由行**：不属于任何 1:1 对的行

一个合法的合并组合（原文侧为例）需同时满足：

- **连续**：所有行在原文中连续
- **恰好包含一个基准行**：保证每个组合天然绑定到唯一 1:1 对，天然去重
- **至少 2 行**：单行不成组
- **不超过 θ 行**：默认 θ = 20

译文侧的合并组合同理。N:M 不存在——系统不引入自己不能保证的新信息。

### 3.4 Phase 4 — 批量编码与评分

收集所有合并组合，拼接文本后通过 `CachedEncoder` 批量编码。以 `content_hash(拼接文本)` + `"_"` + `模型名` + `"_"` + `instruction_hash` 为键存入 SQLite，跨文档共享。

**为什么必须拼接后重新编码？** 取多行嵌入向量的平均再算余弦会抹去语序和递进关系——"门开了。她走进来。"取平均后因果链消失。

### 3.5 Phase 5 — 单一 DP 最终决选

| 操作类型 | 候选来源               | 评分方式           |
| -------- | ---------------------- | ------------------ |
| 1:1      | 真锚点 + 赝锚点        | 预计算的余弦相似度 |
| N:1      | Phase 3 原文侧合并组合 | Phase 4 批量编码   |
| 1:M      | Phase 3 译文侧合并组合 | Phase 4 批量编码   |
| 1:0      | 未覆盖的原文自由行     | 恒为 0.0           |
| 0:1      | 未覆盖的译文自由行     | 恒为 0.0           |

> 1:0 和 0:1 得分恒为 0.0——**不设惩罚**。承认缺失，比假装知道更安全。

## 四、数学假设

### H1 — 行级单调映射

若原文行 i 对应译文行 j，则对任意 i' > i，其对应 j' ≥ j。不可补偿的前提——段落重排将使对齐彻底失败。

### H2 — 容器连贯性

连续孤行构成语义整体，应打包为 N:0 或 0:M 而非散落碎片。

### H3 — 双侧 Top-1 邻域归位（核心假设）

正确翻译对在源侧和目标侧的相似度排名都在可预测区间内。不要求余弦值忠实反映语义距离，只要求排名行为可预测。

### H4 — 得分退化线性模型

低分区域的信任余量线性缩小——信任余量与 top-1 得分呈正比。

### H5 — 零惩罚模型

1:0 和 0:1 是有效状态而非错误，不设惩罚。承认缺失比假装知道更安全。

## 五、质量门控

### G1 — 真锚点密度

当 `anchor_density < 60%` 时标记为 `unreliable`。

真锚点密度定义：

$$
\text{anchor\_density} = \frac{2 \times \text{true\_anchor\_count}}{\text{src\_line\_count} + \text{tgt\_line\_count}}
$$

其中 `true_anchor_count` 为 Phase 1 递归搜索产出的**纯真锚点对数**（不含 Phase 2 补全的赝锚点）。每对真锚点涉及 1 条原文行 + 1 条译文行，且递归 LIS 保证行级不重复。

**示例**：原文 100 行、译文 100 行，算法产出 40 对真锚点 → 密度 = 80 ÷ 200 = 40%。

> 注意：此处的"真锚点"仅指 Phase 1 找到的高置信度锚点，Phase 2 的赝锚点不计入密度计算，以避免低分候选稀释质量指标。

### G2 — 孤行占比

当 `gap_row_ratio ≥ 10%` 时追加 `gap_dominated` 标记。

### G3 — 合并触顶

当 `n_overflow_rows > 0` 时追加 `merge_overflow` 标记，表示合并组合数量超过了枚举上限。

## 六、Instruction 机制

**此机制仅对支持 Instruction 前缀的嵌入模型生效。** 当前推荐使用 Qwen3-embedding 系列或基于其微调的模型（如默认的 `leoipulsar/harrier-0.6b`），它们在预训练阶段原生支持 Instruction 引导，能充分利用此增强。通用嵌入模型（如 nomic-embed-text 等）未经过 Instruction 训练，前缀不会带来增益——可在 GUI 设置面板中通过开关关闭。

> **配置方式**：GUI 设置面板 → 嵌入模型 Tab → 勾选「启用 Instruction 前缀」并填入文本。也可通过环境变量 `DUALIGN_INSTRUCTION` 设置（适用于 CLI / 无 GUI 场景）。关闭方式：取消勾选，或将文本留空，或将环境变量设为空字符串。

默认 Instruction 文本：

```text
Instruct: Identify parallel sentences across languages
Query:
```

编码时自动拼接在每句文本前，使嵌入空间更关注平行/非平行的语义区分，而非文本表面的风格特征。

> 以下基线基于内部约 1 万行轻小说语料摸底。完整验证需在不同语言对和文档类型上系统复现。

### H1 — 单调性

- 预期：违例数为 0（DP 回溯正确性检验）

### H3 — 语义分离（Cohen's d）

- 正确配对分布：均值 ≥ 0.75，标准差约 0.06
- 干扰候选分布：均值 0.60–0.68
- Cohen's d ≥ 1.5（摸底结果 1.93）

### Instruction 增益

- 有/无 Instruction 的对角线均值差 Δ ≥ 0.10

### 信任余量有效性

- 高度平行语料上 Phase 1 与受限 DP 共识度 > 99%
- trust margin 的真实价值体现在非逐句对应数据（如字幕）
